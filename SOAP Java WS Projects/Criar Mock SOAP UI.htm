Criando um mock no SOAP UI sem um WSDL de base.

 

1-     Crie um arquivo XML com o conteúdo abaixo

2-     Se desejar troque a porta de 8888 pela desejada e a url de acesso Mock, antes de seguir os próximos passos.

 

 

<?xml version="1.0"?>

<definitions name="Mock"

      targetNamespace="http://localhost:8888/Mock.wsdl"

      xmlns:tns="http://localhost:8888/Mock.wsdl"

      xmlns:xsd1="http://localhost:8888/Mock.xsd"

      xmlns:soap="http://schemas.xmlsoap.org/wsdl/soap/"

      xmlns="http://schemas.xmlsoap.org/wsdl/">

 

  <!-- Request e Response -->

  <message name="MockOperationInput"></message>

  <message name="MockOperationOutput"></message>

 

  <!-- Cria uma operacao de exemplo -->

  <portType name="MockPortType">

    <operation name="MockOperation">

      <input message="tns:MockOperationInput"/>

      <output message="tns:MockOperationOutput"/>

    </operation>

  </portType>

 

  <!-- NOME DO BINDING -->

  <binding name="MockSoapBinding" type="tns:MockPortType">

    <soap:binding style="document" transport="http://schemas.xmlsoap.org/soap/http"/>

    <operation name="MockOperation">

      <soap:operation soapAction="http://localhost:8888/MockOperation"/>

        <!-- MANDA UM LITERAL E DEVOLVE QUALQUER VALOR -->

      <input><soap:body use="literal"/></input>

      <output><soap:body use="literal"/></output>

    </operation>

  </binding>

 

  <!-- URL DE ACESSO AO SERVICO -->

  <service name="MockService">

    <documentation>SOAP UI Mock modelo</documentation>

    <port name="MockPort" binding="tns:MockSoapBinding">

      <soap:address location="http://localhost:8888/Mock"/>

    </port>

  </service>

 

</definitions>

 

 

3  Importe o XML acima no SOAP UI.

 

 

 

4  Gere  um Mock Service clicando com o botão direito em cima do Binding.

 

5  Vai abrir uma janela, altere a PATH e a PORTA para a URL correta da sua REQUEST, deixas idênticas, como na imagem abaixo.

 

 

 

 

6  Agora vamos fazer um teste, inicie seu Mock Service que vai informar (running on port 8888).

            E roda uma request, ela deve aparece no Message Log do MockService, informando que recebeu e respondeu a requisição.

 

 

7  Agora basta apagar/editar/replace todo o conteúdo do response pelo conteúdo que vc deseja, e rodar uma nova request.

            Ao mudar o response não precisa parar ou restartar o mockservice, basta alterar e requisitar novamente.
